2
環境変数（.env）とは？
APIキーなど重要な情報を安全に管理する仕組み
1

📄 .env ファイルに変数を書く

APIキーなどを「変数名 = 値」の形で記録しておく。

2

📞 他のファイルから変数名で呼び出す

長いキーを直接書かなくても、変数名だけで使える。コードがスッキリする。

3

🙈 GitHubには絶対に上げない

.gitignore に追加してリモートに反映されないよう隠す。

# .env ファイルの中身（イメージ）
GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXX
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_KEY=eyJhbGciOiJXXXXXXXX

# 他のファイルでの呼び出し（Python の例）
api_key = os.environ["GEMINI_API_KEY"]
⚠️

GitHubに上げると何が起きる？
リポジトリが公開設定だと世界中に見られる。APIキーは「そのサービスへのパスワード」なので、悪用されると自分のアカウントで勝手に使われて課金される。

📋

チーム開発のコツ： .env 本体は隠しつつ、.env.example（中身はダミー値）をGitHubに上げておく。メンバーがどの変数が必要かわかるテンプレートになる。

🧪 Python の仮想環境（venv）について

プロジェクトごとに「専用の部屋」を作ってインストールする慣習がある。やらないと別プロジェクトのライブラリと混ざって不具合が起きる。

😵

venv なし

全プロジェクトのライブラリがPCに混在→バージョン衝突・謎のエラー

VS
😌

venv あり

プロジェクトごとに独立した部屋→クリーンな状態を維持できる

# 仮想環境を作成して入る（バックエンド担当向け）
python -m venv venv
source venv/bin/activate  # Mac / Linux
venv\Scripts\activate    # Windows
