基本のき

フロントとバックって何が違うの？

「表側」と「裏側」—— レストランに例えるとイメージしやすい。

🍽️
フロントエンド
ホールスタッフ
お客さん（ユーザー）が直接見て・触れる部分。見た目と操作を担当する。
ボタン・テキスト表示 画面の切り替え 入力フォーム デザイン・レイアウト

👨‍🍳
バックエンド
キッチン
ユーザーには見えない裏側。データの処理・保存・取得を担当する。
ログイン認証 データをDBに保存 AIへのリクエスト パスワード照合

よくある勘違い

「ボタンを押したら画面が変わる」はどっちの仕事？

これ、実はフロントエンドの仕事。バックエンドは「データ」だけを扱う。

ボタンを押したら画面が変わる → バックエンド
→
画面の変化・表示切替 → フロントエンドの仕事
FastAPI = フロントとバックをつなぐ橋
→
FastAPIはバックエンドそのもの。橋じゃなくてキッチン本体。
GeminiやClaudeが「自分たちのバックエンド」
→
外部サービス。FastAPIから呼び出す「別のお店のキッチン」。

実際の処理の流れ

ログインボタンを押したとき、何が起きてる？

フロントとバックがAPIを通じてやりとりする様子。

フロントエンド（Next.js）
ユーザーがボタンを押す → 入力されたIDとパスワードを受け取る
画面の操作はすべてフロントが担当
API通信
「このIDとパスワードで合ってる？」とバックに送る
これが「APIを通じたやりとり」
バックエンド（FastAPI）
DBを照合して「OK」か「NG」を返す
データ処理・保存・照合はバックが担当
フロントエンド（Next.js）
結果を受け取り、OKならホーム画面に切り替える
「画面を変える」最後の判断もフロントがやる

言語とフレームワーク

JavaScriptとNext.jsは何が違う？

各行をクリックすると詳しい説明が見られる。

言語
JavaScript
ブラウザが理解できる基本の言葉。これだけでもWebは作れる。

料理で言う「包丁・フライパン」。これ自体が道具であり、素材でもある。

素のJavaScriptだけで画面を作ると、こうなる：

document.getElementById('btn') .addEventListener('click', function() { document.getElementById('text') .textContent = '押された' // 画面の変化が増えるほどこれが増殖する… })
ライブラリ
React
UI部品（コンポーネント）を作って使い回せる仕組み。JSで書かれてる。

「部品を一回定義して何度でも使う」という発想。レシピカードを100個表示するとき、定義は1回でいい。

function RecipeCard({ name, time }) { return ( <div> <h2>{name}</h2> <p>{time}分</p> </div> ) } // 使うときはこれだけ（何個でも） <RecipeCard name="肉じゃが" time={30} />
フレームワーク
Next.js
Reactをベースに、よく使う機能を全部入れてくれた道具箱。

Reactだけだと自分で設定しないといけないことが多い。Next.jsはそれを最初から用意してくれてる。

// ページ遷移：フォルダ置くだけで自動対応 pages/ index.tsx → / recipes/ [id].tsx → /recipes/123 // API通信の仕組みも込み // 表示速度の最適化も自動
まとめると： JavaScriptという言語だけでフロントは作れる。でも現実的な規模のアプリを作るには、その言語を使いやすくした道具（React・Next.js）が必要。言語が変わるわけじゃなくて、上に積み重なっていくイメージ。
