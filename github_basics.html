チーム開発の必須スキル

GitHub、
地図で覚える。

「どこからどこへ動くか」で考えると、コマンドが自然にわかってくる。

なぜ使うのか

GitHubが必要な2つの理由

👥

複数人でコードを書くため

チームで同時に開発するとき、誰がどこを直したか管理できる。互いの作業が衝突しないように整理してくれる。

📸

履歴を残すため

変更のたびに「写真」を撮るイメージ。壊してしまっても、過去の状態にいつでも戻せる安全網になる。

基本操作

6つの操作をひとつずつ

01
💾
コミット
ローカル内 ↺ 変更を記録

変更をローカルに「セーブ」する操作。写真を1枚撮るイメージ。何をしたかをメッセージで残すのが大事。

# 変更内容をひとまとめにして記録
git add .
git commit -m "ログイン画面を修正"
02
🚀
プッシュ
💻 ローカル → ☁️ リモート

ローカルのコミットをGitHub（リモート）に送る操作。「押し出す」イメージ。チームに自分の変更を共有できる。

# 自分のブランチをGitHubに送る
git push origin feature/login
03
📥
プル
☁️ リモート → 💻 ローカル

リモート（GitHub）の最新状態を自分のPCに取り込む操作。「引っ張ってくる」イメージ。毎朝作業前に実行するのが基本。

# チームの最新コードを自分のPCに取得
git pull origin main
04
🙋
プルリクエスト
feature/〇〇 → main

「自分のブランチをmainに入れてもいいですか？」とチームに申請する操作。GitHub上でレビューしてもらい、OKが出たらマージ。

# GitHub画面で "New Pull Request" を作成
# コマンドではなくウェブ上の操作
→ レビュー → 承認 → マージ
05 / 06
🌿
ブランチ

mainから分岐した「作業用の写しのスペース」。ここで作業することでmainを壊さずに開発できる。

# ブランチを作って移動
git checkout -b feature/login
🔀
マージ

featureブランチで作った変更をmainブランチに「合流」させる操作。PRが承認されると実行される。

# featureの変更をmainに合体
git merge feature/login
ブランチとマージの流れ
main
feature/login（ここで作業）
mainのコミット
featureのコミット
マージポイント
+ clone
📋
クローン（最初の1回だけ）
☁️ リモート → 💻 ローカル（新規）

チームに参加したとき・新しいPCで始めるときなど、GitHubにあるリポジトリを自分のPCにまるごとコピーしてくる操作。プルと違い「最初の1回だけ」使う。

git clone https://github.com/チーム名/リポジトリ名.git

実際の流れ

1日の作業ルーティン

1
git pull origin main
最新状態を取得する
作業前に必ずプル。誰かが昨日プッシュしていたら、その変更を自分のPCに取り込む。
2
git checkout -b feature/新機能
作業用ブランチを作る
mainを直接触らず、作業用の分岐を作る。名前は「feature/何をするか」が慣例。
3
コードを書く
実装する
区切りのいいタイミングで次のコミットへ。一気にやりすぎず、こまめにセーブするのがコツ。
4
git add . && git commit -m "〇〇を実装"
変更をコミットする
ローカルにセーブ。メッセージは「何をしたか」を具体的に書く。「修正」だけだと後で意味がわからなくなる。
5
git push origin feature/新機能
GitHubに送る
ローカルのコミットをリモートへ。これでチームメンバーがコードを見られる状態になる。
6
GitHub上でPull Requestを作成
レビューを依頼する
「mainに入れていいですか？」と申請。チームにチェックしてもらう。
7
Merge Pull Request
マージして完了
レビューOKならマージ。featureブランチの内容がmainに合流し、チーム全員のコードに反映される。

よくある疑問

つまずきやすいポイント

⚡

コンフリクトが起きたら？

同じ行を別々の人が編集してマージしようとすると起きる。Gitが「どっちを使う？」とファイルに書いてくれるので、選んで保存するだけ。怖くない。

🌿

ブランチ名はどうする？

「feature/ログイン機能」「fix/ボタンの色」のように「種類/内容」で付けるのが慣例。英語で書くことが多いが、チームで統一できれば何でもOK。

🔄

プルとクローンの違いは？

クローンは「初回だけ」リポジトリ全体を丸ごとコピーする操作。プルは「2回目以降」の差分だけを取り込む操作。
